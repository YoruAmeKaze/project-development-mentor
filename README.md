# 项目开发教练

##### 一步步教你部署互联网项目的AI教练，分步骤，有条理，易上手。

## Skill简介

本Skill主要面向编程初学者、大学生和Vibe Coding用户，意在帮助用户写代码，做项目的同时，教授用户相对应的技术和知识。会在开发过程中：

- 分析项目本质
- 拆解开发流程
- 规划开发顺序
- 推荐合适技术栈
- 控制项目复杂度
- 解释每一步为什么这样做
- 在关键阶段提供代码实现

本Skill的目标不是代替用户开发项目，而是帮助用户将自己的想法变为现实，并且理解项目中的每一处技术点。

##  核心能力

### 1.项目分析与拆解

分析这个项目的核心基础功能
帮助用户：

- 找出 MVP（最小可行产品）
- 区分核心功能与后期功能
- 防止项目一开始过度设计

- 判断项目真正的难点

例如：

用户想做“校园二手平台”时， Skill 会优先推荐：

- 登录系统
- 商品发布
- 商品列表

而不是一开始就做：

- 即时聊天
- 推荐算法
- 微服务架构

### 2. 真实开发顺序规划

Skill 会按照真实工程开发流程推进项目。

例如：

1. 项目初始化
2. 数据库设计
3. 用户系统
4. API 开发
5. 前端页面
6. 权限系统
7. Docker 部署

而不是随机生成功能代码。

### 3. 分阶段教学式开发

每次只推进一个阶段。

Skill 会：

- 解释当前阶段目标
- 说明为什么先做这个
- 告诉用户需要什么技术
- 提供当前阶段代码
- 等待用户输入“下一步”后再继续

避免新手被大量代码淹没。

### 4.工程思维与复杂度控制

Skill 会主动提醒：

- 哪些功能现在不该做
- 哪些技术对当前阶段没必要
- 哪些地方最容易踩坑
- 为什么某些架构不适合新手项目

例如：

- 不会一开始推荐 Kubernetes
- 不会强行上微服务
- 不会为了“高级感”增加复杂度

## 使用场景

### 编程课程项目

帮助第一次做课程设计的学生：

- 理清项目结构
- 规划开发顺序
- 理解代码逻辑

### Web 全栈项目开发

适合：

- 博客系统
- 电商平台
- 管理后台
- 校园平台
- AI 工具网站

### Vibe Coding 辅助开发

当用户已经有项目想法，但不知道：

- 从哪里开始
- 先写什么
- 技术怎么选
- 项目怎么拆

Skill 会像真正的开发导师一样逐步推进。

### 如何使用

推荐对话起手式：

我想做一个校园二手平台

或者：

我想做一个 AI 工具网站

或者：

我正在学习 FastAPI，想做一个练手项目

Skill 会自动：

1. 分析项目本质
2. 拆解 MVP
3. 规划开发顺序
4. 推荐技术栈
5. 开始第一阶段开发

在每个阶段结束后：

- 总结当前完成内容
- 提示下一阶段
- 等待用户输入“下一步”后继续

## 注意事项

### 不会一次性生成完整大型项目

Skill 的目标是帮助用户真正理解开发流程，而不是瞬间生成无法理解的大型代码库。

### 优先推荐简单稳定方案

对于新手项目：

优先：

- 单体架构
- SQLite / MySQL
- FastAPI / Flask
- Vue / React

避免过早引入复杂系统。

### 强调真实工程顺序

不会出现：

- 后端还没写就先部署
- 登录没完成就开始做高级功能
- MVP 没完成就提前优化架构

## 总结

很多 AI Coding 工具的问题不是“不会生成代码”。

而是：

它们默认用户已经理解完整工程开发流程。

但对于大多数新手来说，真正困难的其实是：

- 不知道从哪里开始
- 不知道先做什么
- 不知道为什么这样设计
- 不知道项目怎么一步一步完成

所以我做了这个 Skill。

希望它更像：

“一个真正带你做项目的开发导师”。
